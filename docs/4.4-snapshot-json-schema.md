# snapshot_json Schema

The `games.snapshot_json` column holds a game's entire authoritative state as
one JSON document (see [4.2 Database Schema](4.2-database-schema.md#games) for
why it's authoritative and how the other columns/tables are just projections of
it). This doc describes that document's shape — the thing every other column is
derived from and the only thing `load_game` reads back.

It's derived directly from `crates/server-game/src/persistence.rs`'s
`PersistedGame` (the serialized type) and the structs it nests from
`crates/server-game/src/game_state.rs`. If this drifts from those, the code
wins — re-derive it from there rather than trusting stale prose here.

## Why a serialized document, not columns

The runtime game object (`GameSession`) is a rich graph — a 15×15 board, the
shuffled tile bag, each seat's *hidden* rack, full move history, chat, turn
state. It's `serde`-serialized whole to this blob on save and deserialized
straight back on load (SQLite used as a document store for the game aggregate).
Two payoffs: a **lossless** round-trip (columns can't represent racks / bag
order / board, so only the blob can reconstruct a game — hence "authoritative")
and **decoupling** (adding a field to the game model is a `#[serde(default)]`,
not a schema migration).

`PersistedGame`/`PersistedVariantRules` are deliberately *separate* types from
the runtime `GameSession`/`VariantRules` they mirror, so the on-disk shape stays
stable while the domain types are free to change, with the conversions in
`persistence.rs` keeping up.

## Top level: `PersistedGame`

| Field | Type | Notes |
|---|---|---|
| `id` | string | Game id (UUID v4). |
| `status` | `"waiting"` \| `"active"` \| `"finished"` | Mirrors `games.status`. |
| `variant` | string | Edition name (`official`, `wordfeud`, `north_american`, `german`, `spanish`). |
| `language` | string | Dictionary/language (`sowpods`, `german`, `spanish`, …). |
| `board_layout` | string | Board layout name. |
| `rules` | `PersistedVariantRules` | Per-game frozen ruleset (see below). `#[serde(default)]` → `official` for snapshots predating per-game rules. |
| `turn_number` | i64 | Increments each turn; used for the move-reminder gate. |
| `current_seat` | u8 | Seat whose turn it is (0-based). |
| `winner_seat` | u8 \| null | Set on a single clear winner; null for a tie or unfinished. |
| `final_bonus_seat` | u8 \| null | Seat that got the end-of-game unplayed-tiles bonus, if any. `#[serde(default)]`. |
| `final_bonus_points` | i32 \| null | That bonus's size. `#[serde(default)]`. |
| `creator_player_id` | string \| null | Who created the game (may hold no seat, e.g. a Bot Showdown). `#[serde(default)]` — null on pre-field snapshots. |
| `removed_by_creator` | bool | Creator hid this (finished) game from their own list. `#[serde(default)]`. |
| `random_seed` | u64 | RNG seed the bag was shuffled with at creation. |
| `board` | array of `BoardCellDto` | The 225 board squares (letter/blank/empty + premium), see [4.3 API Schema](4.3-api-schema.md). |
| `bag` | array of `Tile` | Remaining tiles in the bag, in draw order. |
| `participants` | array of `ParticipantState` | One per seat (see below). |
| `moves` | array of `MoveRecord` | Full move history, in order (see below). |
| `messages` | array of `ChatMessageRecord` | In-game chat, in order. `#[serde(default)]` → `[]` on pre-chat snapshots. |
| `consecutive_scoreless_turns` | u8 | Drives the scoreless-turn-limit game end. `#[serde(default)]`. |
| `move_time_limit_seconds` | u64 | Per-turn auto-retire limit (a *duration*, not a timestamp — stays u64). `#[serde(default)]` → the 72h default. |
| `turn_started_at` | i64 | Unix seconds the current turn began (drives move-time-limit + reminders). `#[serde(default)]` → now; string-or-number tolerant on load (see [Forward compatibility](#forward-compatibility)). |

## `PersistedVariantRules`

The edition's frozen ruleset, stored per game so a game keeps playing under the
rules it was created with even if an edition's definition later changes.

| Field | Type | Notes |
|---|---|---|
| `name` | string | Edition name. |
| `language` | string | Dictionary/language. |
| `letter_values` | array of u8 | Point value per letter index. `Vec` (not fixed array) so a game from before the alphabet widened past 26 still loads. |
| `tile_distribution` | array of u8 | Tile count per letter index, same `Vec` reasoning. |
| `alphabet` | `Alphabet` | `Single(char)` / `Double(char, char)` graphemes (Spanish CH/LL/RR). `#[serde(default)]` → `latin26` on pre-field snapshots. |
| `blank_tiles` | u8 | Blank-tile count. |
| `rack_size` | u8 | Tiles per rack. |
| `width`, `height` | u8 | Board dimensions. |
| `bingo_bonus` | `Score` | Bonus for using the whole rack. |
| `premiums` | array of `Premium` | The 225 square premiums (blank/DL/TL/DW/TW). |

## `ParticipantState` (one per seat)

| Field | Type | Notes |
|---|---|---|
| `seat_number` | u8 | Turn-order index (0-based). |
| `kind` | `"human"` \| `"engine"` | |
| `display_name` | string | |
| `player_id` | string \| null | Claiming human's id; null for an engine or unclaimed seat. |
| `engine_id` | string \| null | Assigned engine id; null for a human seat. |
| `score` | i32 | Current total score. |
| `rack` | `Rack` | This seat's tiles — the hidden state that only the blob captures. |
| `resigned` | bool | Seat has exited (resigned / forced / timed out); stays visible but inactive. |
| `removed_by_player` | bool | This seat's player hid the finished game from their own list. Per-viewer only. `#[serde(default)]`. |
| `invited_email` | string \| null | For an `Email` seat claim, until claimed. `#[serde(default)]`. |
| `reminder_sent_turn` | i64 \| null | The `turn_number` a move-reminder email was last sent for on this seat. `#[serde(default)]`. |

## `MoveRecord` (one per move, in order)

| Field | Type | Notes |
|---|---|---|
| `move_number` | i64 | Sequential index within the game. |
| `seat_number` | u8 | Who moved. |
| `move_type` | string | `place` \| `pass` \| `exchange` \| `resign` \| `force_resign` \| `timeout` \| `admin_force_end`. |
| `main_word` | string \| null | The main word formed by a `place`, if any. |
| `score_delta` | i32 | Points this move scored (0 for non-scoring moves). |
| `description` | string | Human-readable line ("Alice played CARROT for 24", "Bob resigned"). |
| `positions` | array of `PositionDto` | Squares a `place` used; empty for other move types. `#[serde(default)]`. |

## `ChatMessageRecord` (one per chat message, in order)

| Field | Type | Notes |
|---|---|---|
| `id` | string | UUID v4, generated client-side when sent. |
| `player_id` | string | Sender. |
| `display_name` | string | Sender's name at send time. |
| `body` | string | Message text. |
| `created_at` | i64 | Unix seconds sent; string-or-number tolerant on load (see below). |

## Forward compatibility

The blob is deserialized with a hard `.expect("persisted game should parse")`,
so a snapshot that fails to deserialize takes its game down. Two patterns keep
old snapshots loading:

- **Additive fields** use `#[serde(default)]` (or `default = "fn"`): a field
  added after some snapshots were written is simply absent in those, and
  defaults on load. This covers every field marked "`#[serde(default)]`" above
  and is the normal, zero-friction way to extend the model.
- **A reshaped field** — an existing field whose *type* changed — is the hard
  case `serde` can't default, and has bitten this project before (the
  `Alphabet` change, and the `timestamps → i64` change that added this doc).
  `turn_started_at` and `ChatMessageRecord.created_at` use
  `deserialize_with = "game_state::deserialize_i64_flexible"`, which accepts the
  old string form *or* the new number and yields `i64`; the next save rewrites
  it as a number. **Any future change that reshapes a type reachable from
  `PersistedGame` needs a shim like this** (a custom `Deserialize`/
  `deserialize_with`, or a versioned old/new enum) — otherwise it silently
  breaks every existing game.
